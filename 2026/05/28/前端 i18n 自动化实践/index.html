我们现在管理后台的项目在处理 i18n 时，整个流程是比较繁琐的。
项目里的翻译文件由 TMS 系统统一维护，前端通过 pnpm lang 拉取最新翻译文件。但新增文案时，中间有很多重复的人工操作：

如果每次都靠人工复制、翻译、整理，很容易浪费时间，也容易出错。

所以就想做一套自动化方案，把最机械的部分自动化掉。

具体流程：
通过 Node 脚本扫描源码中的中文，调用阿里云机器翻译生成英文译文，自动产出可导入 TMS 的 Excel，最后再拉取 TMS 翻译并替换源码中的中文文案。

翻译方案的选择

最开始考虑过几种方案：

第三方 google-translate-api 虽然不需要 key，也不用绑定云服务，但它属于非官方方案，容易限流或失效，更适合临时本地工具，不太适合作为团队流程的一部分。

Google 官方 API 稳定，但开通时需要绑卡和预付款，实现起来比较麻烦。
