前言

最近开始自学 Java，准备在前端基础上继续往全栈方向发展，并把学习过程整理记录下来。

目前已经完成的学习内容：

  • 确认公司的 Java 技术栈

一、明确学习目标

现阶段希望先做到：

  1. 能看懂 Java 后端项目
  2. 能本地启动和调试 Spring Boot 项目
  3. 能理解一个接口从请求到数据库的完整流程
  4. 能修改简单接口
  5. 能独立完成基础 CRUD 模块
  6. 后续能够参与公司 Java 项目的实际开发

所以不需要按照传统路线，花几个月把 Java 基础、JVM、多线程、Spring 全部学完，再开始做项目。

八、初步理解后端项目结构

1. Controller：接口层

Controller 用来接收前端请求。

例如前端发送一个登录请求：

请求会先进入对应的 Controller 方法。

Controller 通常负责：

  • 定义接口地址
  • 接收请求参数
  • 进行基础参数校验
  • 调用 Service
  • 返回统一格式的数据

可以简单理解为，Controller 是后端接口的入口。

2. Service：业务层

Service 负责具体的业务逻辑。

例如一个登录接口，可能需要完成：

  1. 根据用户名查询用户
  2. 判断用户是否存在
  3. 校验密码
  4. 创建登录状态
  5. 生成 Token
  6. 返回登录结果

这些逻辑一般不会全部写在 Controller 中，而是交给 Service 处理。

这样做可以让接口层和业务层职责更加清晰。

3. Mapper：数据访问层

Mapper 负责和数据库交互。

项目使用 MyBatis-Plus，所以很多基础数据库操作不需要手写完整 SQL。

例如根据主键查询：

可以简单理解为，Mapper 就是用来操作数据库的，负责执行增删改查等操作。

4. Entity：数据库实体

Entity 是用来表示数据库数据的 Java 类。通常一张表对应一个 Entity 类，表中的字段对应类的属性，表中的一行数据对应一个 Entity 对象。

例如，数据库中有一张 user 表，在 Java 中可以定义对应的 User 类：

查询数据库时，Mapper 会将查询结果转换成 User 对象：

在 Java 代码中，数据库中的一条记录会用一个 Entity 对象来表示。

5. DTO：数据传输对象

DTO 是用来传递接口数据的，用于接收请求参数或者组织返回结果。

例如，登录时前端只需要提交用户名和密码：

后端可以定义一个对应的 DTO：

当前端提交 JSON 请求时，Spring Boot 会通过内置的消息转换器，将请求数据转换成 LoginRequestDTO 对象，Controller 可以直接读取其中的用户名和密码，不需要让前端提交完整的 User 实体。

这样做的好处是：

  • 接口字段更明确
  • 避免前端传入无关字段
  • 请求对象和数据库实体解耦
  • 后续接口调整更方便

十、后续学习规划

1. 补齐 Java 基础

学习内容

  • 变量、String、条件判断、循环、方法和数组
  • 类与对象、构造方法、接口、集合、泛型和异常
  • package 和 import

项目练习

完成目标：看到一个简单方法时，能够说清楚它接收什么参数、做了什么、调用了谁，以及返回什么结果

2. 理解 Web 和数据库基础

学习内容

  • HTTP 请求、URL、GET、POST、PUT、DELETE、JSON、请求参数、请求体和常见状态码
  • 数据库、表、字段、主键，以及 SQL 的增删改查

项目练习：沿着登录接口理解“前端提交 JSON → Controller 接收数据 → Mapper 查询 MySQL → 返回结果”的过程

3. 学习 Spring Boot 和项目常用技术

学习内容

  • Spring Boot 常用注解、依赖注入和 application.yml
  • Maven、Lombok、参数校验、全局异常处理和 Sa-Token
  • MyBatis-Plus 的查询、增删改、条件查询和分页

完成目标：修改一个已有接口，并独立增加一个简单接口

4. 独立完成公告管理 CRUD

练习模块：以 notice 公告管理为例

需要完成

  • 从数据库表开始编写 Entity、DTO、Mapper、Service 和 Controller
  • 补上参数校验、统一返回和接口测试
  • 实现分页列表、详情、新增、编辑、删除和启用/禁用接口

完成目标：独立写出一组可运行的 CRUD 接口，而不是只修改现有代码

5. 完成前后端联调和权限控制

前后端联调

  • 实现公告管理页面，包括分页列表、标题搜索、新增和编辑表单、删除和状态切换
  • 处理 Token 携带和接口错误提示

权限控制

  • 使用 Sa-Token 学习登录拦截、获取当前用户、接口权限、角色权限、401、403 和退出登录
  • 例如，未登录用户不能新增公告，普通用户只能查看，管理员才能编辑和删除

完成目标

6. 按业务需要补充常用能力

完成一个模块后，再按实际需求补充：

  • Redis 缓存
  • 文件上传
  • 操作日志
  • 定时任务
  • GitLab CI

例如，可以缓存公告详情、上传公告附件，或定时发布公告。

7. 最后学习完整后台和微服务

完整后台系统

  • 学习 SmartAdmin、RuoYi-Vue-Plus 等项目
  • 重点关注用户、角色、菜单、数据权限、字典和代码生成

微服务组件

Spring Cloud、Nacos、Gateway、Dubbo、RocketMQ、Seata、Elasticsearch 和 XXL-Job 放在最后。开始这些内容前，先确保能够熟练完成普通 Spring Boot 接口和数据库操作。

技术 主要作用
Spring Cloud 提供服务注册、配置管理、路由等微服务常用能力
Nacos 服务注册与发现、配置管理
Gateway 统一接收和转发前端请求的网关
Dubbo 服务之间的 RPC 调用
RocketMQ 异步消息传递
Seata 保证多个服务之间的数据一致性
Elasticsearch 全文搜索和数据分析
XXL-Job 分布式定时任务
