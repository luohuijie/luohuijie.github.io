简介

Redux 是 JavaScript 状态容器，提供可预测化的状态管理。Redux不耦合任何框架，在React、Vue、Angular、jQuery等都可以使用。

为什么使用Redux呢？

  • 随着项目的越来越庞大和复杂，组件之间状态难以维护。
  • Redux是为了解决state的数据管理难问题。
  • 在React开发中由于单项数据流，两个非父子组件通讯麻烦。

Redux工作原理

简述上图的一个工作：

ComponentA派发一个动作dispatch(action)（动作意思是改变state颜色为红色），reducer接受到这个动作后改变state颜色，然后通知所有的订阅者，ComponentB和ComponentC监听到state有变化，拿到最新的state，改变组件颜色为红色。

  • store是应用的集中管理的地方，也可以称之为仓库。
  • state里保存我们的数据。
  • reducer处理器，接受到动作之后修改state状态。
  • dispatch派发一个动作通知reducer要修改的state，然后循环调用缓存函数，subscribe回掉函数触发拿到新状态。

action是一个普通对象，描述想要进行什么操作。组件修改state，必须要通过Reducer处理器，不能直接操作state。

总结

Redux 是一个用于管理应用状态的工具，它通过一个全局的 store 来存储应用的状态，并通过 action 来触发状态的改变。Redux 的核心概念包括 action、reducer 和 store。

  • action 是一个普通的 JavaScript 对象，它描述了发生了什么。
  • reducer 是一个纯函数，它接收当前的状态和 action，并返回新的状态。
  • store 是一个 JavaScript 对象，它保存了应用的状态，并提供了一些方法来获取和更新状态。

Redux 的优点：

  1. 管理应用状态变得简单和可预测。
  2. 状态的变化可以被追踪和调试。
  3. 状态的变化可以被预测和测试。

Redux 的缺点：

  1. 学习曲线较陡。
  2. 需要编写更多的代码。
  3. 对于小型应用来说可能过于复杂。

Redux是一个强大的工具，可以帮助我们更好地管理应用状态。但也有一些缺点，需要根据具体的应用场景来决定是否使用Redux。
