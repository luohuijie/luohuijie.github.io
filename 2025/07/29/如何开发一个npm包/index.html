8. 发布到 NPM

首先，确保你有一个 npm 账户。如果没有，请在 npm 官网 注册一个。
然后，在命令行中登录你的 npm 账户，输入你的用户名、密码和邮箱。

在运行 npm login 命令之前，必须先把下包的服务器地址切换为 npm 的官方服务器。否则会导致发布包失败！

登录成功后，就可以发布包了：

  • 默认情况下，当你使用 npm publish 命令发布一个包时，npm 会根据你的包的配置和你在 npm 注册表中的设置来决定包的访问权限。
  • 如果你的包是在一个组织下，并且该组织的默认访问权限是私有的（private），那么使用 npm publish 将会把包发布为私有包，只有组织成员可以访问。

发布成功后，你可以在 npm 官网 上找到你的包。

12. 删除已发布的包

运行 npm unpublish 包名 –force 命令，即可从 npm 删除已发布的包

  • npm unpublish 命令只能删除 72 小时以内发布的包
  • npm unpublish 删除的包，在 24 小时内不允许重复发布
  • 发布包的时候要慎重，尽量不要往 npm 上发布没有意义的包

13. 发布到其他 npm 仓库

如果你需要发布到私有 npm 仓库，可以使用 npm publish --registry <registry-url> 命令，其中 <registry-url> 是你的私有 npm 仓库的地址。

最佳实践建议

  • 使用语义化版本控制（SemVer）：1.0.0 → 1.1.0 → 2.0.0
  • 添加 CI（GitHub Actions）验证构建 & 测试
  • 加入 ESM + CommonJS 双构建支持（使用 rollup等工具进行构建）
  • 添加 TypeScript 类型定义（如使用 tsc）
  • 添加单元测试（如使用 vitest）
  • 添加 README.md 文件，说明如何安装、使用、测试和贡献你的包
  • 添加 LICENSE 文件，说明你的包的许可证
  • 发布前使用 npm pack 检查发布文件包内容
  • 使用 changesets 管理多包发布或自动发版

总结

自定义 npm 包的需求和功能非常广泛，可以涵盖从工具库、UI 组件、状态管理到性能优化、监控分析等多个方面。根据项目的具体需求，开发者可以灵活选择和设计自定义包，以提升开发效率和代码质量。

本文介绍了如何开发一个 npm 包，包括创建项目、编写代码、添加单元测试、配置 TypeScript、构建、发布到 NPM 等步骤。希望对你有所帮助！
