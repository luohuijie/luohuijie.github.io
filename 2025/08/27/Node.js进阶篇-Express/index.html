前言

Express 是一个基于 Node.js 的 Web 应用框架，它可以帮助我们快速地构建 Web 应用和 API。在本文中，我们将学习如何使用 Express 创建一个简单的 Web 应用，并介绍一些常用的 Express 功能和中间件。

一、什么是 Express？

  • Node.js 本身可以用来写 Web 服务，但是它的 http 模块比较底层，需要自己处理路由、请求解析、响应等逻辑。
  • Express 是一个基于 Node.js 的轻量级 Web 框架，帮我们封装了很多常用功能，比如：
    • 路由系统（根据 URL 和请求方法来分发处理函数）
    • 中间件机制（请求 → 一系列中间件处理 → 响应）
    • 视图渲染（支持 EJS、Pug 等模板引擎）
    • 静态资源服务（直接托管 CSS、JS、图片等）

一句话总结：Express 可以更高效地构建 Web 应用和 API 服务。

五、Express 中间件

1. 什么是中间件？

中间件就是处理请求的”中转站”，是 Express 的核心机制：

  • 请求到达服务器 → 经过一系列中间件处理 → 返回响应。

特点：

  • 一个中间件处理完后需要调用 next() 才会继续下一个
  • 可以对请求和响应进行加工

常见场景：

  • 记录日志
  • 解析请求体（body）
  • 权限验证
  • 错误处理

十二、项目结构优化

随着功能越来越多，把所有逻辑都写在 app.js 会很混乱。推荐的结构：

express-demo/
├── app.js          # 项目入口
├── routes/         # 路由
│   ├── user.js
│   └── todo.js
├── controllers/    # 控制器（处理业务逻辑）
│   └── todoController.js
├── models/         # 数据模型（数据库）
├── middlewares/    # 自定义中间件
├── public/         # 静态资源
└── views/          # 模板文件

这种方式更适合扩展，遵循 MVC 思想。
